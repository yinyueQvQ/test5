# AI图像分割与风格迁移应用

这是一个基于SwiftUI的iOS应用，集成了图像分割和动漫风格迁移功能，使用了两个CoreML模型：
- `is-net-genral-use.mlmodel` - 用于图像分割
- `animeganPaprika.mlmodel` - 用于动漫风格迁移

## 功能特点

### 🎯 主要功能
1. **图像选择** - 支持从相册选择或相机拍摄
2. **AI图像分割** - 自动识别并分割图像中的不同区域
3. **区域选择** - 用户可以选择特定的分割区域
4. **风格迁移** - 将选中区域转换为动漫风格
5. **结果保存** - 将处理后的图片保存到相册

### ✨ 特色功能
- **彩色区域显示** - 不同颜色代表不同的物体区域
- **交互式选择** - 点击选择需要处理的区域
- **实时预览** - 实时查看选中区域的效果
- **流程指示器** - 清晰的步骤指引
- **一致的分辨率** - 所有图像统一处理为512x512分辨率

## 应用流程

```
选择图片 → 图像分割 → 区域选择 → 风格迁移 → 查看结果
    ↓           ↓           ↓           ↓           ↓
   📱         ✂️          👆          🎨          ✅
 拍照/相册   AI自动分割   选择目标区域   动漫风格转换   保存/重新开始
```

## 技术架构

### 核心组件

1. **ImageSegmentationManager** 
   - 负责图像分割处理
   - 管理`is-net-genral-use.mlmodel`模型
   - 生成彩色分割区域
   - 提供区域选择和提取功能

2. **StyleTransferManager**
   - 负责风格迁移处理  
   - 管理`animeganPaprika.mlmodel`模型
   - 将图像转换为动漫风格

3. **ImagePickerView**
   - SwiftUI与UIKit的桥接
   - 支持相机和相册访问

4. **ImageProcessingUtils**
   - 图像处理工具类
   - 提供标准化、格式转换等功能
   - 内存管理和错误处理

### UI设计

- **现代化界面** - 使用SwiftUI构建，支持深色模式
- **渐变色按钮** - 美观的视觉效果
- **卡片式布局** - 清晰的信息层次
- **动画效果** - 流畅的用户体验
- **响应式设计** - 适配不同屏幕尺寸

## 安装与使用

### 系统要求
- iOS 14.0+
- Xcode 12.0+
- 支持CoreML的设备

### 安装步骤

1. **克隆项目**
   ```bash
   git clone [项目地址]
   cd test5
   ```

2. **确保模型文件存在**
   - `test5/is-net-genral-use.mlmodel`
   - `test5/animeganPaprika.mlmodel`

3. **配置权限**
   - Info.plist已包含相机和相册访问权限
   - 首次使用时会请求用户授权

4. **构建运行**
   - 在Xcode中打开`test5.xcodeproj`
   - 选择目标设备
   - 点击运行

### 使用指南

1. **选择图片**
   - 点击"从相册选择"或"拍摄照片"
   - 选择要处理的图像

2. **等待分割**
   - AI自动分析图像并生成分割结果
   - 不同颜色代表不同的物体区域

3. **选择区域**
   - 点击色彩卡片选择需要的区域
   - 可以多选，实时预览选中效果

4. **风格迁移**
   - 点击"开始风格迁移"
   - 等待AI处理完成

5. **查看结果**
   - 预览最终效果
   - 可以保存到相册或重新开始

## 技术细节

### 图像处理
- **标准化尺寸**: 所有图像统一处理为512x512像素
- **颜色空间**: 使用RGB颜色空间
- **内存优化**: 自动清理临时资源
- **质量保证**: 高质量重采样算法

### 模型集成
- **CoreML框架**: 原生iOS机器学习支持
- **Vision框架**: 图像分析和处理
- **异步处理**: 避免UI阻塞
- **错误处理**: 完善的异常捕获机制

### 性能优化
- **懒加载**: 按需加载模型和资源
- **内存管理**: 自动释放不需要的资源
- **并发处理**: 后台线程执行AI推理
- **缓存机制**: 避免重复计算

## 注意事项

1. **设备性能** - AI处理需要一定的计算资源，建议在较新的设备上运行
2. **网络连接** - 应用本地运行，无需网络连接
3. **存储空间** - 模型文件较大，确保有足够的存储空间
4. **隐私保护** - 所有图像处理都在本地完成，不会上传到服务器

## 故障排除

### 常见问题

**Q: 分割结果不理想？**
A: 尝试选择光线良好、对比度高的图片，避免过于复杂的背景

**Q: 风格迁移效果不佳？**
A: 确保选中的区域包含明显的物体轮廓，避免选择纯背景区域

**Q: 应用运行缓慢？**  
A: 检查设备剩余内存，关闭不必要的后台应用

**Q: 无法访问相机/相册？**
A: 检查系统设置中的隐私权限，确保已授权应用访问

### 错误代码
- `InvalidImage`: 图像格式无效或损坏
- `ProcessingFailed`: 图像处理失败，请重试  
- `ModelNotLoaded`: AI模型加载失败
- `UnsupportedFormat`: 不支持的图像格式
