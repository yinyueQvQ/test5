# 🔌 无线使用配置指南

## 问题：为什么连数据线能用，不连就不行？

### 连着数据线时（能用）：
- ✅ Xcode 通过 USB 调试
- ✅ 可能使用端口转发访问 localhost
- ✅ 可以访问局域网地址

### 不连数据线时（不能用）：
- ❌ 无法访问 `127.0.0.1` (localhost)
- ❌ 无法访问 `192.168.x.x` (局域网)
- ✅ **需要使用公网地址（ngrok）**

```swift
// ❌ 使用了 http
private let publicEndpoint = "http://abc123-def.ngrok-free.app/generate"

// ❌ 多了空格
private let publicEndpoint = "https://abc123-def.ngrok-free.app /generate"
```

## ⚠️ 常见问题

### 问题 1：ngrok URL 过期

**症状：**
- 之前能用，现在不能用了
- 错误：`Connection refused` 或 `404`

**原因：**
- ngrok 免费版每次启动 URL 都会变
- 关闭终端后 ngrok 停止运行

**解决：**
1. 重新启动 ngrok
2. 复制新的 URL
3. 更新 Swift 代码
4. 重新编译 App

---

### 问题 2：连接超时

**症状：**
- 请求发出但一直等待
- 最后显示 `timeout`

**可能原因：**
1. ngrok 没有运行（检查终端）
2. Flask 服务器没有运行
3. URL 配置错误（缺少 `/generate`）

**排查：**
```bash
# 检查 ngrok 进程
ps aux | grep ngrok

# 检查 Flask 进程
ps aux | grep python
```

---

### 问题 3：端口冲突

**症状：**
- Flask 启动失败
- 提示 `Address already in use`

**解决：**
```bash
# 查找占用 8000 端口的进程
lsof -i :8000

# 杀掉进程
kill -9 <PID>

# 重新启动 Flask
python3 server_api_remote.py
```

## 📊 两种模式对比

### 连数据线模式（调试用）

**优点：**
- ✅ 速度快（局域网）
- ✅ 不需要 ngrok
- ✅ 可以实时调试

**缺点：**
- ❌ 必须连数据线
- ❌ 不方便移动

**适合：** 开发和调试阶段

---

### 无线模式（正式使用）

**优点：**
- ✅ 无需连数据线
- ✅ 可以自由移动
- ✅ 更接近真实使用场景

**缺点：**
- ❌ 速度稍慢（通过公网）
- ❌ 需要启动 ngrok
- ❌ 每次启动 ngrok 都要更新 URL

**适合：** 测试和演示

---

## 🎯 快速切换

如果你需要经常切换连线/无线模式：

### 方式 1：修改代码切换

```swift
private var apiEndpoint: String {
    return localEndpoint   // 连数据线时用这个
    // return publicEndpoint  // 无线时用这个
}
```

每次切换后需要重新编译 App。

---

### 方式 2：动态配置（需要添加代码）

可以在 App 设置里添加一个开关，让用户选择使用哪个服务器。

---

## 🔧 ngrok 免费版限制

**免费版限制：**
- 每次启动 URL 都会变
- 每月有连接数限制
- 可能需要在浏览器确认（点击 "Visit Site"）

**升级到付费版可以：**
- 固定 URL（不用每次改代码）
- 无连接数限制
- 更快的速度
