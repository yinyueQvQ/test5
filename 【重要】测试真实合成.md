# 🚨【重要】测试真实合成

## ❗ 为什么服务器没有输出？

### 原因：你可能一直在用**预设组合**测试！

预设组合会直接返回本地图片，**不会调用服务器API**，所以：
- ✅ Xcode终端有输出（显示"检测到xxx组合"）
- ❌ 服务器终端没有输出（根本没发请求）
- ✅ 手机显示了图片（预设的精美图片）

---

## 📋 预设组合列表

以下组合**不会调用服务器**，会直接返回预设图片：

1. 🌟 **灯泡** + **星星**
2. 🌸 **杯子** + **花**
3. 🍋 **柠檬** + **瓶子** 或 **柠檬** + **树叶**
4. 🛁 **浴球** + **花**
5. 🌺 **瓶子** + **花**
6. ❄️ **雪花** + **星星**

**注意：** 这些名称匹配是模糊的，只要素材名称中包含这些关键词就会触发！

例如：
- "灯泡素材" + "星星图片" → ✅ 会触发预设
- "素材灯泡123" + "我的星星" → ✅ 会触发预设

## 🐛 仍然没有输出？

### 检查清单

#### 1. 确认API端点

查看日志中的：
```
📍 API 端点: https://percurrent-sub-lavette.ngrok-free.dev/generate
```

**必须包含 `/generate` 后缀！** 我已经修复了这个问题。

---

#### 2. 确认ngrok正在运行

在ngrok终端应该看到：
```
Forwarding  https://percurrent-sub-lavette.ngrok-free.dev -> http://localhost:5000
```

**如果URL变了，需要更新 `StableDiffusionManager.swift` 中的 `publicEndpoint`！**

---

#### 3. 确认服务器正在运行

在服务器终端运行：
```bash
curl https://percurrent-sub-lavette.ngrok-free.dev/health
```

应该返回：
```json
{"status":"ok"}
```

---

#### 4. 查看完整错误日志

如果请求失败，Xcode控制台会显示：
```
❌ 网络错误: ...
❌ 无响应数据
❌ HTTP 状态码: 404/500/...
```

把错误信息告诉我！

## 📊 快速测试步骤

1. **打开Xcode控制台**（View → Debug Area → Activate Console）

2. **在App中选择2个素材**（名称不包含：灯泡、星星、杯子、花、柠檬、瓶子、树叶、浴球、雪花）

3. **点击"开始合成"**

4. **立即查看Xcode控制台**
   - 看到 "✅✅✅ 没有匹配任何预设组合" → ✅ 正确

5. **如果正确，继续看**
   - 看到 "🌐 ========== 开始 API 调用 ==========" → ✅ 请求已发送
   - 看到 "❌ 图片合并失败" → ❌ 图片有问题

6. **查看ngrok和服务器终端**
   - 有请求日志 → ✅ 成功
   - 没有日志 → 网络问题
