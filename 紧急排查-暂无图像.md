response = requests.post(url, json=payload, timeout=300)
print(f"📡 状态码: {response.status_code}")

if response.status_code == 200:
    data = response.json()
    print(f"✅ 成功！返回了 {data.get('count', 0)} 张图像")
else:
    print(f"❌ 失败：{response.text}")
EOF

**如果这个测试成功**：说明服务器正常，问题在网络或 App

## 📊 可能的原因

### 原因 1：请求超时 ⏱️

**症状：**
- Xcode 显示：`❌ 网络错误: The request timed out`
- 服务器可能还在生成中

**解决：**
- ✅ 我已增加超时到 300 秒
- 禁用 HuggingFace 联网（节省 30 秒）

---

### 原因 2：生成失败 💥

**症状：**
- 服务器显示：`✗ 生成失败`
- 输出目录是空的

**可能原因：**
- 提示词有问题（特殊字符）
- 模型文件损坏
- 内存不足

**排查：**
```bash
# 在服务器上检查
nvidia-smi  # 查看显存使用
free -h     # 查看内存使用
```

---

### 原因 3：传输失败 📡

**症状：**
- 服务器显示生成成功
- 但 App 显示"无响应数据"

**可能原因：**
- 图片太大（base64 编码后）
- 网络不稳定
- Flask 响应超时

**解决：**
检查生成的图片大小：
```bash
# 在服务器上
ls -lh /root/stablediffusion-main/outputs/img2img-samples/samples/*.png
```

如果图片 > 5MB，需要压缩。
