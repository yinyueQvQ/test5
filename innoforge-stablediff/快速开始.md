# 🚀 快速开始 - Stable Diffusion 远程调用

## 你的情况

- ✅ 输入图片在本地 Windows 电脑
- ✅ Stable Diffusion 项目在远程服务器
- ✅ 需要从本地发送图片 → 服务器生成 → 下载结果

---

## 📝 三步快速开始

### 第 1 步：安装依赖（2 分钟）

在本地 Windows 电脑上打开 PowerShell 或命令提示符：

```bash
cd D:\innoforge-stablediff
pip install paramiko
```

### 第 2 步：测试连接（1 分钟）

运行连接测试脚本：

```bash
python test_connection.py
```

**期望输出**：
```
✓ SSH 连接成功！
✓ SFTP 连接成功！
✓ 找到项目目录: /root/stablediffusion
```

如果看到错误，检查：
- 服务器地址和端口是否正确
- 用户名和密码是否正确
- 网络连接是否正常

### 第 3 步：配置并运行（5 分钟）

#### 3.1 修改配置

用文本编辑器打开 `client_sd_remote.py`，找到第 250 行左右的配置区域：

```python
# ============ 配置区域 ============
# 服务器配置（已经填好了，如果需要可以修改）
SERVER_CONFIG = {
    "host": "connect.westc.gpuhub.com",
    "port": 42742,
    "username": "root",
    "password": "hHI9fR1QZ8g/"
}

# 项目路径配置
# ⚠️ 重要：根据 test_connection.py 的输出修改这里
SERVER_PROJECT_DIR = "/root/stablediffusion"  # 修改为实际路径

# 本地配置
# ⚠️ 重要：修改为你的实际图片路径
LOCAL_INPUT_IMAGE = r"D:\stablediffusion-main\data\data_final\light.jpg"
```

#### 3.2 运行生成

```bash
python client_sd_remote.py
```

#### 3.3 等待完成

脚本会自动：
1. 连接到服务器
2. 上传你的图片
3. 执行生成命令
4. 下载生成的图片到本地

**生成的图片保存在**：`generated_images/` 目录

## 🎯 你需要修改的地方

只需要修改 `client_sd_remote.py` 中的 **3 个配置**：

### 1. 服务器项目目录

```python
SERVER_PROJECT_DIR = "/root/stablediffusion"  # ← 改成你的实际路径
```

**如何找到正确路径？**
- 运行 `python test_connection.py`，它会自动检测
- 或者 SSH 登录服务器：`ssh -p 42742 root@connect.westc.gpuhub.com`
- 然后运行：`ls -la` 查看目录

### 2. 模型路径（可选）

```python
CHECKPOINT_PATH = f"{SERVER_PROJECT_DIR}/ckpt/512-base-ema.ckpt"
```

如果模型文件名不同，修改 `512-base-ema.ckpt` 为实际文件名。

### 3. 本地输入图片

```python
LOCAL_INPUT_IMAGE = r"D:\stablediffusion-main\data\data_final\light.jpg"
```

改成你的实际图片路径。

### 4. 提示词（可选）

```python
PROMPT = """你的提示词..."""
```

## 📋 常见问题速查

### ❌ 连接失败

**错误**: `Connection timeout` 或 `Connection refused`

**解决**:
```bash
# 测试 SSH 是否可以连接
ssh -p 42742 root@connect.westc.gpuhub.com

# 如果能连接，说明配置正确
# 如果不能连接，检查网络和服务器状态
```

### ❌ 找不到模型文件

**错误**: `FileNotFoundError: checkpoint not found`

**解决**:
```bash
# SSH 登录到服务器
ssh -p 42742 root@connect.westc.gpuhub.com

# 查看模型目录
ls /root/stablediffusion/ckpt/

# 复制实际的模型文件名，修改脚本中的 CHECKPOINT_PATH
```

### ❌ 项目目录不存在

**错误**: `No such file or directory: /root/stablediffusion`

**解决**:
```bash
# SSH 登录服务器
ssh -p 42742 root@connect.westc.gpuhub.com

# 查找项目目录
find ~ -name "stablediffusion" -o -name "stable-diffusion"

# 或者
ls -la ~

# 找到正确路径后，修改脚本中的 SERVER_PROJECT_DIR
```

### ❌ 命令执行失败

**错误**: `Command failed with exit code 1`

**解决**: 查看脚本输出的错误信息，常见原因：
- 输入图片格式不支持（尝试转换为 JPG）
- 服务器磁盘空间不足
- CUDA/GPU 错误

可以手动 SSH 登录测试命令：
```bash
cd /root/stablediffusion
python scripts/img2img.py --help  # 查看帮助
```

### ❌ 下载目录为空

**错误**: 下载完成但没有图片

**解决**:
```bash
# SSH 登录服务器，检查输出目录
ls /root/stablediffusion/outputs/samples/

# 如果目录不存在或路径不对，需要修改脚本
# 或者检查生成命令的输出参数
```

## 🎨 修改生成参数

在 `client_sd_remote.py` 中找到命令构建部分（约第 180 行）：

```python
command = f'cd {server_project_dir} && python scripts/img2img.py'
command += f' --prompt "{prompt}"'
command += f' --init-img "{remote_input_file}"'
command += f' --ckpt "{checkpoint_path}"'

# 👇 可以添加更多参数
# command += ' --n_samples 4'        # 生成 4 张图片
# command += ' --ddim_steps 50'       # 50 步采样
# command += ' --scale 7.5'           # CFG scale
# command += ' --strength 0.75'       # img2img 强度
```
