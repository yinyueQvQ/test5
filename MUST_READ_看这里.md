# ⚠️ 必读！如何查看 Xcode 控制台

## 🚨 重要：如果看不到日志，请按照以下步骤操作

### 第1步：打开 Xcode 控制台（**必须做！**）

#### 方法1：使用快捷键（最简单）
```
同时按下这三个键：
Command (⌘) + Shift (⇧) + Y
```

#### 方法2：点击按钮
在 Xcode 窗口的**右上角**，找到三个图标：
```
┌─┐  ┌─┐  ┌─┐
│ │  │ │  │▓│  ← 点击最右边这个（像个窗口分割）
└─┘  └─┘  └─┘
```

#### 方法3：使用菜单
```
顶部菜单栏 → View → Debug Area → Show Debug Area
```

### 第2步：确认控制台可见

打开后，Xcode 窗口应该是这样的：

### 第3步：清空旧日志

在控制台内：
1. **右键点击**（或双指点击触控板）
2. 选择 **"Clear Console"**
3. 控制台变空白

### 第4步：运行应用

1. 停止当前运行：按 `Command + .`（点号）
2. 重新运行：按 `Command + R`
3. **等待应用启动**

### 第5步：在应用中操作

1. 进入合成台
2. **选择2个素材**（绿色边框）
3. 选择工艺风格
4. **点击"开始合成"**

### 第6步：立即查看控制台

## ⚠️ 如果还是看不到任何输出

### 可能原因1：控制台被过滤了

在控制台**底部**，找到搜索框：
```
┌──────────────────────────────────────┐
│  控制台内容...                       │
│                                      │
└──────────────────────────────────────┘
[🔍 搜索框]  [下拉菜单: All Output ▼]
```

1. 清空搜索框
2. 确保下拉菜单选择 **"All Output"**

### 可能原因2：应用没有连接到调试器

重新启动应用：
1. `Command + .` 停止
2. `Command + R` 运行
3. 看控制台是否有 "Build succeeded" 等信息

### 可能原因3：控制台分割太小

拖动控制台边缘，把它**拉高一些**！

## 🎯 确认清单

在点击"开始合成"之前：
- [ ] Xcode 控制台已打开（Command + Shift + Y）
- [ ] 控制台可见（底部有黑色区域）
- [ ] 旧日志已清除（右键 → Clear Console）
- [ ] 应用正在运行（能看到界面）
- [ ] 已选择2个素材（绿色边框）

如果以上全部 ✅，点击"开始合成"后**立即**看控制台！
